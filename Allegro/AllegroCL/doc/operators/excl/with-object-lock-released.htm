with-object-lock-released

Arguments: &body body

This macro can only appear lexically inside the body of a with-locked-object macro. The lock obtained in the immediately enclosing with-locked-object macro is released, body is executed, and then the lock is reobtained. When the enclosing with-locked-object macro specifies a non-smp value, the with-object-lock-released cooperates to manipulate the corresponding state. This macro isn't allowed in a non-smp lisp if the enclosing lock form designates :atomically as the non-smp strategy. with-object-lock-released is rarely used, and is never actually needed; there are alternate ways of achieving the same thing that are usually clearer.

See smp.htm and multiprocessing.htm for more information on this macro and on multiprocessing.
