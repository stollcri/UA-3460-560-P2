font-pixel-height

Arguments: font

This is a tentative function that likely is not useful. On Windows it simply returns what font-size returns. On GTK, it returns what GTK calls the "ink height" of the font, which is the height that encompasses the actual pixels that are drawn for a line a text, rather then the amount of space that is reserved for each line of text (which includes the "leading" of the font).

In versions of GTK earlier than 2.6, CG is unable to determine the real pixel size of a font due to all requested font point sizes being multiplied by an unknown global "font dots-per-inch" setting. This function might be useful in determining how large a font actually turned out to be in GTK 2.4. In GTK 2.6, CG can specify a font size in pixels directly (which fits with the general CG design of specifying sizes in pixels), and so this is no longer an issue.
