jnotify-lisp

Arguments: notifier handle event longs strings

This method simulates an event arriving as if from the JavaLinkCommon.ltoj_anchor.callLisp() notifier method in Java lightweight event handlers. The effect is to schedule a call to a handler registered with jregister-handler.

the arguments are:

  • notifier is the value of (jlinker-slot :notifier) (see jlinker-slot).
  • handle is 0 or some value returned by jregister-handler.
  • event is a string or keyword naming a registered event.
  • longs is the third argument passed to the handler when a signal arrives from Java. It is then a list of integers but when called from Lisp it may be any argument acceptable to the handler.
  • strings is the fourth argument passed to the handler when a signal arrives from Java. It is then a list of strings but when called from Lisp it may be any argument acceptable to the handler.

The timecard example in examples/jlinker/timecard/main.cl uses this method to insure that display updates from Lisp are properly serialized with updates triggered by Java code.

See jlinker.htm for more information on the jLinker facility.
