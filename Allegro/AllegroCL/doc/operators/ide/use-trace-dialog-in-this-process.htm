use-trace-dialog-in-this-process

Arguments:

Causes the current process to direct its trace output to the IDE's Trace Dialog if it is present, rather than to the *trace-output* stream as usual. When the dialog is not present, trace output will still be directed to *trace-output*. There is no exported way to undo this setting.

It would have no effect to call this function in processes that use the Trace dialog anyway. These processes include IDE Listener processes, the IDE GUI process, Run | Run Project processes, and any process that was given the *default-cg-bindings* when it was created.
